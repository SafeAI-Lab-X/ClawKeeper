***\*AI Agent 安全格局分析报告\****

Claude Code 安全特性与 ClawKeeper 融合方案

2026年4月

# **一、背景与目的**

ClawKeeper 是我们自研的 AI Agent 安全框架，目前已具备威胁检测和行为分析能力。为了进一步提升防御水平，我们对市场上主流 AI 编程工具的安全机制进行了调研，重点分析了 Anthropic 的 Claude Code 和 OpenAI 的 Codex。

本报告的目的是：整理 Claude Code 的安全优势，对比它与 Codex 的差异，并说明哪些能力值得融合到 ClawKeeper 中、建议分几个版本逐步落地。

# **二、Claude Code 的安全优势**

Claude Code 是 Anthropic 官方的 AI 编程命令行工具，拥有文件读写、终端命令执行、Git 操作等高权限能力。由于权限面很广，Anthopic 为它建立了一套多层级的安全体系。主要包括以下几个方面。

## **2.1 三层工具权限门控**

每次工具调用前，Claude Code 会过三道关卡：第一层是预配置的自动化规则（比如“读 .md 文件自动放行”“rm -rf 开头直接拒绝”）；第二层是弹窗让用户确认；第三层是用一个独立的 AI 模型分析当前命令是否安全。***\*三道门只要有一道拒绝，操作就不会执行。\****

它还提供了 6 种权限模式，从完全信任到完全拒绝，可以根据使用场景灵活调整。每次审批结果都会记录来源，方便事后审计。

## **2.2 AI 命令分类器**

这是 Claude Code ***\*最有特色的安全设计\****。传统方案主要靠正则表达式匹配命令模式，但容易误报或漏报。例如 rm -rf /tmp/build 其实只是清理构建产物，正则会误报为危险；而 python3 -c “import os; os.system(‘rm -rf /’)” 这种嵌套式的破坏命令，正则又很难覆盖。Claude Code 的 AI 分类器能结合对话上下文理解命令意图，减少误报、提高拦截准确率。

需要注意的是，这个 AI 分类器目前仅限 Anthropic 内部使用，通过 Feature Flag 控制，外部用户无法直接使用。

## **2.3 Bash 命令深度安全检查**

Claude Code 在代码中实现了 ***\*27 项独立的 Bash 命令安全检查\****，覆盖了很多细节场景：命令替换注入（比如 $() 和反引号）、Zsh 特有的危险命令、Shell 变量注入、Unicode 零宽空格混淆等。这些检查从语法层面拦截潜在的恶意命令，比单纯的命令模式匹配更精细。

## **2.4 路径验证与文件保护**

Claude Code 设计了***\*七层路径校验\****，防止通过 tilde 扩展、符号链接跳出、UNC 网络路径等方式绕过安全限制。同时维护了一份危险文件和目录清单（如 .gitconfig、.bashrc、.ssh 等），操作这些路径时会强制要求用户确认。

## **2.5 其他安全特性**

其他值得关注的设计包括：子 Agent 权限隔离（子任务的权限必须低于主任务）、***\*40+ 种密钥泄露模式检测\****（覆盖 AWS、GCP、Azure、GitHub 等平台）、组织级策略覆盖（企业管理员可从云端下发安全策略）、可信设备认证（90 天滚动 Token）、以及 Feature Flag 灰度发布机制。

## **2.6 已知局限**

Claude Code 也有明显的短板：没有内核级沙箱（不用 seccomp、chroot 或容器），完全依赖软件层的权限控制；MCP 服务器可以运行任意代码，需要用户自行判断信任；AI 分类器失败时默认放行（Fail-open），而不是默认拒绝。

# **三、与竞品 Codex 的对比**

OpenAI 的 Codex 是目前市场上另一款主流 AI 编程工具。它的安全思路与 Claude Code 完全不同：Codex 主要靠内核级沙箱做物理隔离，而 Claude Code 主要靠软件层的精细检查和 AI 理解来管控。两者各有优势，但对于 ClawKeeper 这样的外挂式安全框架来说，***\*Claude Code 的多项设计更值得我们借鉴\****。

## **3.1 Claude Code 相对 Codex 的优势**

| ***\*维度\**** | ***\*Claude Code 优势\****                          | ***\*Codex 情况\****               |
| -------------- | --------------------------------------------------- | ---------------------------------- |
| Bash 检查      | 27 项语法级检查，精细到命令替换、变量注入、混淆检测 | 无专门语法检查，依赖沙箱兼底       |
| 密钥泄露检测   | 40+ 种模式全链路扫描                                | 仅 4 类输出脱敏                    |
| 子 Agent 隔离  | 差异化权限 + 工具白名单 + 禁止嵌套                  | 无专门机制                         |
| 竞态防护       | ResolveOnce 原子审批，多信号只生效一次              | 无专门防护                         |
| 权限模式       | 6 种模式，覆盖从开发到锁定全场景                    | 5 种模式                           |
| AI 语义理解    | 结合对话上下文理解命令意图，减少误报                | Guardian AI 也有，但失败时默认拒绝 |

 

## **3.2 Codex 的优势**

Codex 也有 Claude Code 不具备的能力，主要是内核级沙箱（支持 Linux/macOS/Windows 三平台）、四层网络隔离（从内核到代理到防火墙）、以及进程级加固（反调试、禁 core dump、环境变量清洗）。这些是硬隔离能力，即使 Agent 完全失控也无法突破。但***\*这类能力与 ClawKeeper 的外挂式架构不匹配\****，难以直接借鉴。

## **3.3 对 ClawKeeper 的启示**

Claude Code 的优势项——精细的 Bash 检查、广泛的密钥检测、子 Agent 隔离、竞态防护等——***\*都是软件层面的设计，与 ClawKeeper 的外挂式架构天然契合\****。相比之下，Codex 的核心优势在于内核级隔离，这需要控制整个运行环境，不适合我们的场景。因此我们主要从 Claude Code 借鉴能力。



# **四、建议融合路线图**

基于以上分析，我们梳理了 Claude Code 中值得引入 ClawKeeper 的安全特性，建议分两个版本逐步落地。

## **v1.1：基础防御能力**

让 ClawKeeper 具备***\*基本的主动拦截能力\****。

| ***\*特性\****   | ***\*说明\****               |
| ---------------- | ---------------------------- |
| 执行前拦截门     | 工具调用前可阻断执行         |
| 权限持久化       | 支持会话级和永久级权限配置   |
| 输入校验升级     | 接口层结构化校验             |
| 危险文件保护清单 | 受保护路径操作时自动升级风险 |
| Token 预算控制   | 设定使用限额，超限预警       |

 

## **v1.2：检测与管控增强**

***\*补充检测盲区\****，强化细粒度管控。

| ***\*特性\****    | ***\*说明\****                       |
| ----------------- | ------------------------------------ |
| 密钥检测规则扩充  | 补充云平台、AI 平台等 token 模式     |
| 子 Agent 权限隔离 | 子任务自动收紧策略                   |
| 原子审批机制      | 防止并发审批竞态                     |
| 路径验证          | 拦截符号链接跳出、变量注入等绕过手段 |
| Feature Flag      | 新特性渐进式上线，可随时回滚         |
| 组织级策略下发    | 支持云端统一配置安全策略             |

更远期还可以探索 Bash 深度语法检查、MCP 服务器审批流、可信设备认证等方向，视实际进度纳入规划。

# **五、结论**

## **三者定位**

| ***\*系统\**** | ***\*核心定位\****        | ***\*安全思路\****               |
| -------------- | ------------------------- | -------------------------------- |
| Claude Code    | AI 编程工具，内嵌纵深防御 | 多层审批 + 模式检查 + AI 分类    |
| Codex          | AI 编程工具，内嵌硬隔离   | 内核沙箱 + 失败关闭              |
| ClawKeeper     | 独立安全框架，外挂式防护  | 行为画像 + 意图对齐 + 跨会话学习 |

 

## **ClawKeeper 的差异化优势**

ClawKeeper 与前两者的核心区别在于：它是***\*平台无关的独立安全框架\****，不绑定特定 Agent；三层架构（Skill/Plugin/Watcher）可独立部署；拥有 Jensen-Shannon 散度、意图漂移、风险指纹等行为分析能力，这些是 Codex 和 Claude Code 都不具备的。

## **融合 Claude Code 后的预期效果**

如果按上述路线图逐步落地，ClawKeeper 将***\*从“被动检测”迈向“主动防御”\****。v1.1 先解决“能不能拦”的问题，v1.2 再解决“拦得准不准”的问题。整体节奏以可落地为优先，不追求一步到位。
